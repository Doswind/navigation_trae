# 站点导航 - 功能说明书

## 1. 产品概述

**站点导航**是一个面向个人用户的轻量级网址导航应用，采用紧凑、高密度、低留白的视觉风格，帮助用户把常用网站按照自定义分组进行整理，并通过浏览器扩展或独立页面形式快速访问。

### 1.1 核心定位

- 替代浏览器默认新标签页
- 支持多导航页、多分组、多站点管理
- 支持拖拽排序、数据导入导出、主题与语言切换
- 可作为普通网页运行，也可打包为 Chrome / Edge 浏览器扩展

### 1.2 目标用户

- 需要管理大量常用网址的个人用户
- 希望新标签页更加个性化、信息密度更高的用户
- 需要在多台设备或浏览器之间同步导航数据的用户

---

## 2. 页面结构与布局

### 2.1 整体结构

页面由以下区域组成：

1. **顶部导航栏（Header）**：左侧应用标题与图标、中间导航页切换器、右侧功能按钮（编辑、设置、导入、导出）
2. **主内容区（Home）**：以网格形式展示当前导航页下的所有分组卡片
3. **分组卡片（GroupCard）**：包含分组标题和该分组下的站点卡片列表
4. **站点卡片（SiteCard）**：展示站点图标、名称、地址、描述
5. **弹窗层（Modal）**：用于添加/编辑页面、分组、站点，以及设置、导入、导出、确认删除

### 2.2 视觉风格

- **紧凑布局**：小字号、小间距、少留白
- **卡片化设计**：分组和站点均采用圆角卡片
- **响应式网格**：分组数量根据设置和屏幕宽度自动适配
- **主题变量**：通过 CSS 变量控制浅色/深色主题颜色

### 2.3 响应式规则

- 容器宽度：主内容区撑满可用空间，减少两侧留白
- 分组网格：使用 Tailwind 的 `grid-cols-{n}` 实现，支持 1-6 列
- 站点卡片：根据 `cardLayout` 设置切换水平/垂直/紧凑布局
- 动态网格密度：分组卡片内的站点每行显示数量会根据 `groupsPerRow` 自动调整。例如紧凑布局下，1 个分组每行最多显示 12 个站点，2 个分组每行最多 6 个，3 个分组每行最多 4 个，以此类推；小屏幕会自动减少列数，避免卡片重叠

## 3. 导航页管理

### 3.1 导航页概念

导航页是第一级数据容器，每个导航页包含多个分组。用户可以通过导航页切换器在不同导航页之间切换。

### 3.2 导航页切换器（PageSelector）

- **下拉模式（dropdown）**：显示当前页面名称，点击展开下拉菜单选择其他页面
- **标签页模式（tabs）**：以横向标签形式展示所有页面，可直接点击切换
- 当前激活页面高亮显示，非编辑模式下显示为蓝色背景配白色文字
- 每个页面项左侧显示该页面设置的颜色圆点
- **编辑模式**：每个标签页直接显示编辑（铅笔）和删除（垃圾桶）按钮，无需悬停；当前激活页面在编辑模式下同样显示蓝色背景

### 3.3 添加导航页

- 点击导航页切换器中的「添加导航页」按钮
- 弹窗要求输入：
  - 导航页名称（必填）
  - 导航页颜色（可选，用于标识）
- 保存后自动切换到新页面

### 3.4 编辑导航页

- 在编辑模式下，页面标签/下拉项旁显示编辑按钮
- 可修改页面名称和颜色

### 3.5 删除导航页

- 在编辑模式下，页面标签/下拉项旁显示删除按钮
- 删除前弹出确认对话框
- 最后一个导航页不允许删除

### 3.6 导航页数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 页面名称 |
| color | string | 标识颜色（十六进制） |
| groups | Group[] | 分组列表 |

## 4. 分组管理

### 4.1 分组概念

分组是导航页下的二级容器，每个分组包含若干站点卡片。分组以卡片形式展示在主内容区。

### 4.2 添加分组

- 在编辑模式下，主内容区显示「添加分组」按钮
- 弹窗要求输入：
  - 分组名称（必填）
  - 分组颜色（可选，用于标题装饰）
- 保存后分组追加到当前页面末尾

### 4.3 编辑分组

- 在编辑模式下，分组标题右侧显示编辑按钮
- 可修改分组名称和颜色

### 4.4 删除分组

- 在编辑模式下，分组标题右侧显示删除按钮
- 删除前弹出确认对话框
- 分组内的所有站点一并删除

### 4.5 分组排序

- 在编辑模式下，可拖拽分组卡片调整顺序
- 拖拽时显示半透明效果和阴影

### 4.6 分组内站点网格

- 站点网格列数由 `groupsPerRow` 和 `cardLayout` 共同决定
- 紧凑布局：列数最多可达 12 / groupsPerRow，小屏幕自动降级
- 水平布局：列数较紧凑布局少，保证图标和名称不重叠
- 垂直布局：列数最少，卡片横向空间需求大
- 所有布局下站点名称超长时自动截断

### 4.7 分组数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 分组名称 |
| color | string | 标题颜色 |
| sites | Site[] | 站点列表 |

## 5. 站点管理

### 5.1 站点概念

站点是导航的最小单元，对应一个网址。每个站点以卡片形式展示在所属分组内。

### 5.2 添加站点

- 在编辑模式下，分组标题右侧显示「添加站点」按钮
- 弹窗要求输入：
  - 站点名称（必填）
  - 站点地址（必填，需为有效 URL）
  - 站点描述（可选，多行文本框）
  - 图标 URL（可选，可自动获取）
  - 视觉特效（可选，可多选）
- 站点名称和站点地址各自独占一行显示，标签后带红色必填星号
- 保存按钮在必填项为空时禁用
- 保存后站点追加到分组末尾

### 5.3 编辑站点

- 在编辑模式下，鼠标悬停站点卡片显示编辑按钮
- 点击编辑按钮打开编辑弹窗
- 可修改所有字段

### 5.4 删除站点

- 在编辑模式下，鼠标悬停站点卡片显示删除按钮
- 删除前弹出确认对话框

### 5.5 站点排序与跨分组移动

- 在编辑模式下，可拖拽站点卡片调整同一分组内顺序
- 支持跨分组拖拽，将站点从一个分组移动到另一个分组
- 拖拽触发区域：站点名称区域 或 左侧 6 点拖拽图标

### 5.6 站点打开方式

- 非编辑模式下，点击站点卡片在新标签页打开对应网址
- 编辑模式下，点击站点卡片不触发跳转，仅用于拖拽或编辑

### 5.7 站点图标获取

添加/编辑站点时提供以下图标获取方式：

| 服务 | 说明 |
|------|------|
| 智能选择 | 默认选项，并发探测多个服务，使用第一个成功返回的图标 |
| Google | `https://www.google.com/s2/favicons` |
| DuckDuckGo | `https://icons.duckduckgo.com/ip3/{domain}.ico` |
| Clearbit | `https://logo.clearbit.com/{domain}` |
| Icon Horse | `https://icon.horse/icon/{domain}` |
| Favicon Kit | `https://api.faviconkit.com/{domain}/64` |
| Yandex | `https://favicon.yandex.net/favicon/{domain}` |
| 直接获取 | 尝试直接请求目标网站的 `/favicon.ico` |

### 5.8 站点数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 站点名称 |
| url | string | 站点地址 |
| description | string | 站点描述 |
| icon | string | 图标 URL，为空时自动获取 |
| effects | object | 视觉特效配置 |

## 6. 站点卡片视觉特效

每个站点可独立设置以下视觉特效，特效可同时叠加：

### 6.1 高亮显示

- 卡片边框变为金黄色半透明
- 卡片背景添加淡黄色半透明覆盖
- 用于强调重要站点

### 6.2 闪烁效果

- 卡片边框变为红色半透明
- 背景色在浅红和深红之间循环渐变
- 动画时长 1.6 秒，无限循环

### 6.3 跳动效果

- 卡片边框变为蓝色半透明
- 卡片上下跳动，位移范围 0 到 -8px
- 动画时长 1.2 秒，无限循环

### 6.4 抖动效果

- 卡片边框变为紫色半透明
- 卡片左右摇摆，位移范围 -4px 到 +4px
- 动画时长 0.6 秒，无限循环

### 6.5 特效配置字段

```json
{
  "effects": {
    "highlight": true,
    "blink": true,
    "bounce": true,
    "shake": true
  }
}
```

## 7. 编辑模式

### 7.1 进入/退出编辑模式

- 点击顶部「编辑」按钮进入编辑模式
- 编辑模式下按钮变为「完成」，点击退出编辑模式
- 编辑模式状态持久化到当前会话

### 7.2 编辑模式下的变化

- 显示添加分组按钮
- 分组标题右侧显示编辑/删除按钮
- 站点卡片悬停显示编辑/删除按钮
- 站点卡片左侧显示 6 点拖拽图标（悬停时显示），提示卡片可拖动
- 站点名称区域作为拖拽手柄，鼠标移入变为抓取手形，按住即可拖动
- 分组和站点支持拖拽排序

## 8. 设置面板

点击顶部「设置」按钮打开设置面板，包含以下配置项：

### 8.1 导航页展示方式

- 下拉菜单（dropdown）
- 标签页（tabs）

### 8.2 每行显示分组数量

可选 1、2、3、4、5、6 列，默认 4 列。

### 8.3 主题色调

- 浅色（light）
- 深色（dark）
- 跟随系统（system）

切换后立即生效，并持久化到 localStorage。

### 8.4 语言

- 中文（zh-CN）
- English（en）

切换后立即生效，并持久化到 localStorage。

### 8.5 站点卡片显示

可独立开关：

- 显示站点图标
- 显示站点名称
- 显示站点地址
- 显示站点描述

### 8.6 站点卡片布局

- 水平布局（horizontal）：图标在上，名称居中
- 垂直布局（vertical）：图标在左，名称和详情在右
- 紧凑布局（compact）：图标和名称在同一行

### 8.7 字体大小

支持 5 档字体大小，切换后站点卡片和分组卡片的文字、图标、间距会同步缩放：

| 档位 | 名称 | 说明 |
|------|------|------|
| xs | 极小 | 极限高密度显示 |
| sm | 小 | 默认档位，保持紧凑 |
| base | 中 | 适中字号 |
| lg | 大 | 较大字号 |
| xl | 极大 | 最大字号 |

### 8.8 默认配置

| 配置项 | 默认值 |
|--------|--------|
| 主题 | 跟随系统 |
| 导航页展示方式 | 标签页 |
| 每行显示分组数量 | 4 |
| 站点卡片布局 | 紧凑 |
| 字体大小 | 小 |
| 显示站点图标 | 开启 |
| 显示站点名称 | 开启 |
| 显示站点地址 | 关闭 |
| 显示站点描述 | 关闭 |

### 8.9 配置持久化

所有设置项自动保存到浏览器 `localStorage`，下次打开时恢复。

## 9. 数据导入

### 9.1 导入入口

点击顶部「导入」按钮打开导入弹窗。

### 9.2 导入流程

1. 选择导入文件（JSON 格式）
2. 解析文件并在弹窗中显示导入概要
3. 选择导入模式
4. 选择要导入的页面（当文件包含多个页面时）
5. 点击确认导入

### 9.3 导入概要展示

解析文件后显示：

- 文件中的页面列表
- 每个页面的颜色标识
- 每个页面包含的分组数 / 站点数
- 「全选」复选框

### 9.4 导入模式

| 模式 | 说明 |
|------|------|
| 导入到当前页面 | 用文件中第一个选中页面的分组和站点覆盖当前导航页 |
| 作为新页面导入 | 将选中的页面作为新页面追加到现有数据中 |
| 替换所有数据 | 用导入文件完全覆盖所有现有数据 |

### 9.5 支持的文件格式

#### 完整格式

```json
{
  "pages": [
    {
      "id": "page_1",
      "name": "常用导航",
      "color": "#3b82f6",
      "groups": [
        {
          "id": "group_1",
          "name": "搜索引擎",
          "color": "#3b82f6",
          "sites": [
            {
              "id": "site_1",
              "name": "Google",
              "url": "https://www.google.com",
              "description": "全球最大搜索引擎",
              "icon": "",
              "effects": { "highlight": true }
            }
          ]
        }
      ]
    }
  ]
}
```

#### 简化格式

```json
[
  {
    "name": "搜索引擎",
    "color": "#3b82f6",
    "sites": [
      { "name": "Google", "url": "https://www.google.com" }
    ]
  }
]
```

简化格式导入时会自动创建一个默认页面。

### 9.6 导入容错

- 自动为缺失的 `groups`、`sites`、`effects` 字段补默认值
- 自动重新生成页面、分组、站点 ID，避免与现有数据冲突
- 导入完成后持久化到 localStorage

## 10. 数据导出

### 10.1 导出入口

点击顶部「导出」按钮打开导出弹窗。

### 10.2 导出流程

1. 弹窗显示当前所有导航页列表
2. 用户勾选要导出的页面
3. 提供「全选」按钮
4. 点击确认后生成 JSON 文件并自动下载

### 10.3 导出文件格式

导出文件为标准完整格式，包含 `pages` 数组：

```json
{
  "pages": [
    {
      "id": "...",
      "name": "...",
      "color": "...",
      "groups": [...]
    }
  ]
}
```

## 11. 标签栏图标

- 应用提供独立的 SVG 图标（`public/favicon.svg`），显示在浏览器标签栏和扩展工具栏
- 图标为圆角方形蓝色背景，中央是白色简洁指南针图案
- 浏览器扩展构建时自动生成 16/32/48/128 像素的 PNG 版本

## 12. 浏览器扩展

### 12.1 扩展形式

应用可打包为 Chrome / Edge Manifest V3 扩展，替换浏览器默认新标签页。

### 12.2 扩展功能

- 打开新标签页时自动显示站点导航
- 点击工具栏扩展图标可新建标签页并打开导航
- 所有数据通过浏览器 localStorage 保存

### 12.3 扩展构建

```bash
pnpm build:extension
```

产物位于 `dist-extension/` 目录，包含：

- `manifest.json`
- `background.js`
- 图标文件（icon16.png、icon32.png、icon48.png、icon128.png）
- `index.html` 及构建后的 JS/CSS

### 12.4 扩展加载方式

1. 打开 `chrome://extensions` 或 `edge://extensions`
2. 开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择 `dist-extension` 文件夹

## 13. 数据持久化

### 13.1 存储方式

所有应用数据保存到浏览器 `localStorage`：

- `siteNavigator_data`：导航页、分组、站点数据
- `siteNavigator_config`：用户设置
- `siteNavigator_language`：语言偏好
- `siteNavigator_theme`：主题偏好

### 13.2 首次使用

首次打开应用时，如果 localStorage 中没有数据，自动加载示例数据，包含常用搜索引擎、社交媒体、开发工具等分组。

### 13.3 数据备份建议

由于数据仅保存在浏览器本地，建议定期使用导出功能备份数据。

## 14. 技术实现要点

### 14.1 前端技术栈

- React 18 + TypeScript
- Vite
- Tailwind CSS 3
- Zustand（状态管理）
- @dnd-kit/core + @dnd-kit/sortable（拖拽排序）
- lucide-react（图标）

### 14.2 路由

应用内部使用 `HashRouter`，兼容普通网页和浏览器扩展的 `chrome-extension://` 协议。

### 14.3 主题实现

通过 CSS 变量定义主题色，使用 `.dark` 类切换深色模式。系统模式通过 `matchMedia('(prefers-color-scheme: dark)')` 监听。

### 14.4 拖拽实现

- 分组使用 `useSortable` + `SortableContext`
- 站点使用嵌套排序上下文，支持同组排序和跨组移动
- 拖拽时通过 `transform` 和 `transition` 实现平滑动画

## 15. 错误处理与边界情况

### 15.1 导入错误

- 文件格式非法：提示解析失败
- 文件内容为空：提示无数据
- 数据字段缺失：自动补默认值

### 15.2 图标获取失败

- 自动获取失败时使用通用默认图标
- 智能选择模式下依次尝试多个服务

### 15.3 最后一个导航页

- 禁止删除最后一个导航页
- 删除时弹出提示「不能删除最后一个导航页」

### 15.4 URL 处理

- 自动为缺少协议的 URL 添加 `https://`
- 打开站点时使用 `_blank` 并在新标签页打开
